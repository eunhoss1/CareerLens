# CareerLens 메뉴 구성도

CareerLens는 3Depth를 넘기지 않는 구조로 정리합니다.

## 1Depth

```txt
CareerLens
├─ 맞춤채용정보
├─ 커리어 플래너
├─ 지원 관리
├─ 정착 지원
└─ 마이페이지
```

## 상세 구조

```txt
맞춤채용정보
├─ 추천 진단
│  ├─ 프로필 불러오기
│  ├─ 공고 1차 필터링
│  └─ 상위 5개 추천
├─ 추천 결과
│  ├─ 추천 이유
│  ├─ 부족 요소
│  └─ 준비도
└─ 공고 상세 비교
   ├─ 기술
   ├─ 경력
   ├─ 언어
   └─ 포트폴리오

커리어 플래너
├─ 로드맵 목록
├─ 로드맵 상세
│  ├─ 주차별 과제
│  ├─ 준비 항목
│  └─ 완료율
└─ 준비 과제 관리
   ├─ 상태 변경
   ├─ 우선순위
   └─ 보완 자료

지원 관리
├─ 지원 현황
├─ 관심 공고
└─ 서류/포트폴리오 준비

정착 지원
├─ 국가별 체크리스트
├─ 비자/행정 정보
└─ 출국·초기정착 준비

마이페이지
├─ 내 프로필
├─ 해외취업 프로필
│  ├─ 희망 국가
│  ├─ 직무
│  ├─ 경력
│  ├─ 기술
│  └─ 우선순위
└─ 계정 설정
```

## 사용자 흐름

```txt
회원가입/로그인
-> 마이페이지 프로필 입력
-> 맞춤채용정보 추천 진단
-> 커리어 플래너 생성
-> 지원 관리
-> 정착 지원
```

## 메뉴 결정 이유

- `채용공고`는 독립 1Depth로 두지 않고 `맞춤채용정보` 안에 흡수합니다.
- CareerLens의 핵심은 공고 조회가 아니라 사용자 프로필 기반 추천 진단입니다.
- `준비로드맵`은 서비스명 관점에서 `커리어 플래너`로 통일합니다.
- `출국 행정 로드맵`은 지원 이후 흐름까지 담기 위해 `정착 지원`으로 확장합니다.
- `자료실`은 독립 메뉴로 두지 않고 `정착 지원` 하위의 국가/비자/행정 정보로 흡수합니다.
- `검증 배지`는 현재 중간발표 범위에서 제외하고 후순위 확장 기능으로 둡니다.

## 현재 구현 상태

```txt
맞춤채용정보: 구현 중, 추천 진단 화면 운영
커리어 플래너: 구현 중, 로드맵 목록/상세 운영
지원 관리: 화면 placeholder 생성, 기능 예정
정착 지원: 화면 placeholder 생성, 기능 예정
마이페이지: 구현 중, 해외취업 프로필 입력 운영
```
