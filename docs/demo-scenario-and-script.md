# CareerLens 시연 순서 및 발표 대본

## 1. 시연 목표

CareerLens 시연의 핵심은 단순 채용공고 사이트가 아니라,
해외취업 준비 흐름을 데이터 기반으로 연결하는 플랫폼이라는 점을 보여주는 것이다.

시연에서 강조할 문장:

```txt
CareerLens는 사용자의 해외취업 프로필을 공고, 직원 표본, PatternProfile과 비교해 추천 진단을 만들고,
부족 요소를 커리어 플래너 과제로 전환한 뒤,
AI 문서 검증, 지원 관리, 출국/행정/정착 로드맵까지 이어주는 해외취업 준비 플랫폼입니다.
```

## 2. 시연 전 준비

Backend:

```bash
cd backend
mvn spring-boot:run
```

Frontend:

```bash
cd frontend
npm run dev -- --port 3000
```

접속:

```txt
http://localhost:3000
```

데모 계정:

```txt
login_id: demo
password: CareerLens123!
```

AI API를 사용할 경우:

```env
OPENAI_API_KEY=새로_발급받은_키
AI_PLANNER_ENABLED=true
AI_PROVIDER=openai
OPENAI_MODEL=gpt-5.4
```

Duffel 항공 API를 사용할 경우:

```env
TRAVEL_PROVIDER=duffel
DUFFEL_ENABLED=true
DUFFEL_ACCESS_TOKEN=duffel_test_...
DUFFEL_BASE_URL=https://api.duffel.com
DUFFEL_VERSION=v2
```

Amadeus 기존 키를 사용할 경우:

```env
AMADEUS_ENABLED=true
AMADEUS_CLIENT_ID=발급받은_API_KEY
AMADEUS_CLIENT_SECRET=발급받은_API_SECRET
AMADEUS_BASE_URL=https://test.api.amadeus.com
AMADEUS_CURRENCY_CODE=KRW
```

주의:

```txt
API 키는 절대 GitHub, README, PPT 캡처, 카카오톡에 그대로 공유하지 않는다.
```

## 3. 권장 시연 순서

### 1단계. 메인 페이지

화면:

```txt
/
```

설명:

```txt
첫 화면에서는 CareerLens가 해외취업 준비를 공고 추천에서 끝내지 않고,
진단 리포트, 로드맵, 지원 관리, 정착 지원으로 연결하는 플랫폼이라는 점을 보여줍니다.
```

강조 포인트:

- 채용공고 앱이 아니라 해외취업 진단 플랫폼
- 공고/패턴/프로필 기반 추천
- 커리어 플래너와 지원 관리로 연결

### 2단계. 로그인

화면:

```txt
/login
```

설명:

```txt
사용자는 회원가입 또는 로그인을 통해 본인 계정으로 들어갑니다.
이후 마이페이지 프로필, 추천 진단 결과, 로드맵, 지원 기록, 정착 체크리스트가 사용자별로 저장됩니다.
```

### 3단계. 마이페이지 프로필

화면:

```txt
/mypage
```

설명:

```txt
해외취업 추천은 단순히 희망 국가와 직무만 보는 것이 아니라,
기술스택, 경력, 언어 수준, 학력, 포트폴리오, GitHub, 비자 스폰서십 필요 여부,
우선순위 조건까지 함께 저장해서 추천 진단에 사용합니다.
```

강조 포인트:

- 사용자 프로필이 DB에 저장됨
- 같은 사용자가 다시 로그인해도 이어서 사용 가능
- 추천 결과가 입력 프로필에 따라 달라짐

### 4단계. 맞춤채용정보 추천 진단

화면:

```txt
/jobs/recommendation
```

설명:

```txt
CareerLens의 핵심 기능입니다.
공고와 사용자를 직접 비교하는 구조가 아니라,
수동 조사 기반 공고 데이터와 직원 표본을 바탕으로 정리한 PatternProfile을 함께 사용합니다.
사용자 프로필과 공고별 PatternProfile을 비교해 추천 점수, 추천 이유, 부족 요소를 제공합니다.
```

추천 구조 설명:

```txt
사용자 프로필
-> 공고 1차 필터링
-> 공고별 PatternProfile 조회
-> 사용자와 PatternProfile 비교
-> 추천 점수/부족 요소/준비도 산출
-> DiagnosisResult 저장
```

강조 포인트:

- 추천 점수 계산은 AI가 아니라 rule-based
- AI는 설명/로드맵/문서 검증 보조에 사용
- 추천 결과는 DB에 저장됨

### 5단계. 커리어 플래너 생성

화면:

```txt
/planner/[roadmapId]
```

설명:

```txt
추천 결과에서 바로 끝나지 않고, 부족 요소를 주차별 준비 과제로 바꿉니다.
각 과제에는 설명, 기대 산출물, 검증 기준, 예상 시간, 난이도가 포함됩니다.
AI API가 있으면 AI가 과제 초안을 생성하고, 없으면 규칙 기반 과제로 fallback됩니다.
```

강조 포인트:

- 부족 요소가 실행 과제로 전환됨
- 4주/8주/12주 준비 흐름 가능
- 과제 완료 상태 저장 가능

### 6단계. AI 문서 분석

화면:

```txt
/roadmap/employment/documents
```

설명:

```txt
사용자가 만든 산출물을 텍스트, GitHub repository, PDF/DOCX로 제출하면
CareerLens가 과제 기준과 비교해 검증 점수, 강점, 보완점을 제공합니다.
점수 기준을 넘으면 검증 배지가 발급됩니다.
```

강조 포인트:

- GitHub 주소는 개인 프로필이 아니라 실제 repository URL만 허용
- PDF/DOCX 텍스트 추출 가능
- AI 실패 시 rule-based 검증 fallback
- 배지 발급 기준 존재

### 7단계. 검증 배지

화면:

```txt
/mypage/badges
```

설명:

```txt
문서와 프로젝트 검증 결과는 배지로 누적됩니다.
이 배지는 사용자가 실제로 어떤 준비 산출물을 만들었는지 보여주는 증빙 역할을 합니다.
```

배지 기준:

```txt
60점 이상: Bronze
75점 이상: Silver
90점 이상: Gold
GitHub repository 분석 75점 이상: GitHub Project Verified
```

### 8단계. 지원 관리

화면:

```txt
/applications
```

설명:

```txt
커리어 플래너에서 지원 관리로 넘긴 공고를 관심, 지원 준비, 지원 진행 단계로 관리합니다.
단순 상태값이 아니라 마감기한, 로드맵 완료율, 문서 검증 결과를 함께 보여줍니다.
```

강조 포인트:

- 추천에서 실제 지원 준비로 연결
- 공고별 준비 점수
- 서류/GitHub/어학 증빙 체크

### 9단계. 출국로드맵

화면:

```txt
/roadmap/departure
```

설명:

```txt
기업 합격 이후에는 입사 예정일을 기준으로 권장 입국일과 출국 후보 기간을 계산합니다.
Duffel 공식 항공 API 테스트 키가 있으면 항공편 후보도 조회할 수 있고,
키가 없거나 결과가 없으면 규칙 기반 출국 준비 마일스톤만 생성됩니다.
```

강조 포인트:

- 항공권 사이트 크롤링 안 함
- 공식 Amadeus API 사용
- API 실패해도 시연 흐름 유지
- 실시간 가격/예약 보장은 하지 않음

### 10단계. 행정로드맵

화면:

```txt
/roadmap/administration
```

설명:

```txt
비자, 회사 제출 서류, 출국 전 행정 패키지, 입국 후 초기 행정을 하나의 흐름으로 정리합니다.
AI는 비자 조건을 판정하지 않고, 저장된 체크리스트와 사용자 입력을 바탕으로 우선순위만 요약합니다.
```

### 11단계. 정착 지원

화면:

```txt
/settlement
```

설명:

```txt
정착 지원에서는 미국/일본 기본 체크리스트를 사용자별로 생성하고,
준비 전, 진행 중, 완료 상태를 저장합니다.
AI 키가 있으면 체크리스트 상태를 바탕으로 다음 액션을 요약하고,
없으면 규칙 기반 요약을 제공합니다.
```

설명:

```txt
자료실은 해외취업 정보가 파편화되어 있다는 문제를 보완하기 위한 영역입니다.
현재는 미국/일본 중심 수동 검수 자료와 공식기관 확인 링크를 제공하고,
행정로드맵과 정착지원으로 연결됩니다.
```

## 4. 3분 발표 압축 대본

```txt
CareerLens는 해외취업 준비생을 위한 데이터 기반 추천/진단 플랫폼입니다.
일반 채용공고 사이트처럼 공고만 보여주는 것이 아니라,
사용자의 프로필과 공고별 PatternProfile을 비교해 추천 결과와 부족 요소를 제공합니다.

사용자는 로그인 후 마이페이지에 희망 국가, 직무, 기술스택, 경력, 언어, 포트폴리오, 비자 스폰서십 필요 여부를 입력합니다.
추천 진단 화면에서는 이 프로필을 공고와 직무 패턴 데이터와 비교해 추천 공고, 추천 이유, 준비도, 부족 요소를 보여줍니다.

추천 결과는 커리어 플래너로 연결됩니다.
부족 요소는 주차별 준비 과제로 바뀌고,
각 과제에는 기대 산출물과 검증 기준이 포함됩니다.
사용자가 만든 이력서, 포트폴리오, GitHub repository, PDF/DOCX 자료는 AI 문서 분석에서 검증할 수 있고,
검증 점수에 따라 배지도 발급됩니다.

이후 지원 관리에서는 추천 공고를 실제 지원 준비 파이프라인으로 넘겨
마감기한, 서류 준비도, 로드맵 완료율, 검증 결과를 함께 관리합니다.
합격 이후에는 출국로드맵에서 입사 예정일 기준 출국 후보 기간을 계산하고,
Duffel 공식 항공 API와 연동해 항공편 후보를 확인할 수 있도록 확장했습니다.
마지막으로 행정로드맵과 정착지원에서 비자, 서류, 주소 등록, 은행, 통신, 보험 등 초기 정착 준비를 체크리스트로 관리합니다.

현재 단계에서는 외부 공고 크롤링이나 LinkedIn 자동 수집은 하지 않고,
수동 조사와 seed 데이터 기반으로 시연 가능한 프로토타입을 구현했습니다.
AI는 추천 점수 계산이 아니라 로드맵 생성, 문서 분석, 체크리스트 요약 같은 보조 기능에 사용합니다.
```

## 5. 교수님 질문 대응

### Q. 공고 데이터는 자동으로 가져오나요?

```txt
현재 프로토타입에서는 자동 크롤링을 넣지 않았습니다.
LinkedIn이나 채용 사이트 크롤링은 약관과 법적 리스크가 있기 때문에,
지금은 수동 조사/seed 데이터 기반으로 구조를 먼저 검증했습니다.
향후에는 공식 API, 제휴 API, 관리자 검수 기반 입력 시스템을 검토할 계획입니다.
```

### Q. AI가 추천 점수를 계산하나요?

```txt
아닙니다. 추천 점수 계산은 공고, PatternProfile, 사용자 프로필을 비교하는 rule-based 로직입니다.
AI는 설명 생성, 커리어 플래너 과제 생성, 문서 분석, 체크리스트 요약 같은 보조 기능으로만 사용합니다.
```

### Q. 비자 정보는 최신성이 중요한데 괜찮나요?

```txt
비자나 체류자격은 AI가 판정하지 않도록 했습니다.
CareerLens는 공식기관 확인 링크와 체크리스트를 제공하고,
AI는 저장된 체크리스트를 바탕으로 우선순위를 요약하는 역할만 합니다.
최종 판단은 공식기관과 전문가 확인이 필요하다는 고지를 화면에 포함했습니다.
```

### Q. 항공편 API는 어떻게 붙였나요?

```txt
항공사나 여행 사이트를 크롤링하지 않고 Duffel 공식 Flights API를 1순위로 사용하도록 설계했습니다.
프론트가 직접 Duffel을 호출하지 않고, CareerLens 백엔드가 Offer Request를 생성해 항공편 후보를 받아옵니다.
캡스톤 범위에서는 예약이나 결제를 하지 않고 항공편 후보 표시까지만 사용합니다.
키가 없거나 테스트 환경에서 결과가 없을 때도 기존 출국로드맵은 계속 생성되도록 fallback을 두었습니다.
```

## 6. 시연 중 주의

```txt
1. API 키가 화면이나 터미널에 노출되지 않게 한다.
2. Duffel 테스트 환경은 노선/날짜에 따라 결과가 없을 수 있다.
3. AI 응답이 느릴 수 있으므로 fallback 동작을 설명할 준비를 한다.
4. 추천 진단과 플래너 생성은 demo 계정 기준으로 먼저 확인해둔다.
5. PDF/DOCX는 텍스트 기반 파일만 안정적으로 분석된다.
```
