# 메인 페이지 UI 개선 정리

## 작업 목적

CareerLens 메인화면을 기존 서비스 색감과 기능 흐름은 유지하면서, 더 완성도 있는 SaaS 랜딩 페이지 형태로 개선했다. Linear, Vercel, Wanted, Stripe의 정돈된 레이아웃과 제품 미리보기 감각을 참고했지만, 특정 사이트를 복제하지 않고 CareerLens의 해외취업 공고 탐색, 적합도 진단, 준비 로드맵 서비스에 맞게 재구성했다.

## 수정 범위

- `frontend/app/page.tsx`
- `frontend/app/globals.css`
- `docs/main-page-ui-refine.md`

프론트엔드 메인 페이지와 문서만 수정했다. 백엔드, DB, API, 인증 로직, 라우팅 구조는 수정하지 않았다.

## 주요 변경 내용

### Hero 영역

- 메인 문구를 `해외 채용공고 탐색부터 지원 준비까지`로 정리했다.
- 보조 문구는 공고 조건, 프로필, 적합도 진단, 준비 로드맵 연결 흐름이 바로 보이도록 축약했다.
- CTA는 기존 이동 흐름을 유지했다.
  - `적합도 진단 시작` -> `/jobs/recommendation`
  - `전체 공고 보기` -> `/jobs`
- 불필요한 작은 기준 태그와 운영 현황/파이프라인형 UI는 제거했다.
- 첫 화면에서는 Hero 내용만 보이도록 최소 높이를 조정하고, 다음 섹션은 스크롤 후 자연스럽게 진입하도록 했다.
- 밝은 배경, 네이비 포인트, 얇은 border, 부드러운 shadow 중심으로 정돈했다.

### 제품 미리보기

- Hero 오른쪽에 CareerLens 진단 리포트 형태의 제품 미리보기 카드를 배치했다.
- 추천 공고, 직무, 총점, ScoreBar, 부족 요소, 다음 액션, 추가 액션을 하나의 mock dashboard처럼 정리했다.
- 프리뷰 뒤쪽의 보조 박스와 장식 라벨은 제거해 카드 자체만 깔끔하게 보이도록 했다.
- API 연결 방식은 변경하지 않고 기존 정적 UI 흐름을 유지했다.

### 서비스 흐름

- `추천에서 지원까지` 섹션을 5단계로 정리했다.
  - `01 프로필 입력`
  - `02 공고 매칭`
  - `03 적합도 진단`
  - `04 준비 로드맵`
  - `05 지원 관리`
- 각 step card는 번호, 제목, 짧은 설명만 남겨 간결하게 구성했다.
- 데스크톱에서는 5열, 작은 화면에서는 1~2열로 자연스럽게 내려가도록 했다.

### 주요 서비스 카드

- `채용공고`, `맞춤추천`, `적합도 진단`, `준비 로드맵`, `자료실` 5개 카드로 정리했다.
- 각 카드는 라벨, 제목, 짧은 설명, 태그 2~3개로 통일했다.
- 문구는 실제 서비스 화면처럼 기능 중심으로 정리했다.

### 추천 근거 데이터

- `공고 데이터`, `합격자 패턴`, `준비 로드맵`을 밝은 카드 기반 섹션으로 정리했다.
- 딥 네이비를 넓게 사용하지 않고, 브랜드 컬러 progress bar와 얇은 border로 데이터 기반 서비스 느낌을 냈다.

## 인터랙션

- hover 효과 적용 요소:
  - Hero CTA 버튼
  - Hero 통계 카드
  - 제품 미리보기 카드
  - 제품 미리보기 내부 액션 카드
  - 서비스 흐름 step card
  - 주요 서비스 카드
  - 추천 근거 데이터 카드

- scroll 등장 효과 적용 요소:
  - 서비스 흐름 섹션
  - 주요 서비스 섹션
  - 추천 근거 데이터 섹션

- 구현 방식:
  - 새 라이브러리는 추가하지 않았다.
  - 별도 스크롤 컨트롤러는 제거했다.
  - 기존 전역 CSS의 기본 `scroll-rise` 클래스만 유지했다.
  - Tailwind class 기반의 200ms 내외 hover transition을 적용했다.
