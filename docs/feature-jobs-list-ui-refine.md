# 전체 공고 조회 UI 개선 기록

## 목표

CareerLens의 `전체 공고 조회` 화면을 Simplify Jobs의 탐색 경험을 참고해 개선한다. 이번 작업은 프론트엔드 UI와 클라이언트 상태만 수정하며, 백엔드 API, DTO, DB 스키마, 인증 흐름은 변경하지 않는다.

## Simplify Jobs 벤치마킹 관찰

- 검색 결과는 카드 그리드보다 `결과 리스트 + 상세 패널` 구조가 빠른 비교에 유리하다.
- 카드에는 회사 로고, 회사명, 직무명, 고용 형태, 연봉, 위치, 근무 방식처럼 지원 판단에 필요한 정보를 chip 형태로 압축한다.
- 필터는 기본 검색 영역을 작게 유지하고, 추가 조건은 drawer 또는 별도 패널로 묶어 화면 밀도를 낮춘다.
- 상세 화면은 `Overview / Company`처럼 정보를 탭으로 나누고, `Save / Apply` 같은 주요 행동은 하단 또는 상단에 고정한다.
- `Showing N of M`, 최신/마감 상태, 저장 버튼처럼 현재 상태를 계속 노출해 탐색 흐름을 끊지 않는다.

## 적용 방향

- `/jobs` 페이지를 `상단 검색/필터 toolbar + 좌측 결과 리스트 + 우측 상세 패널` 구조로 재구성한다.
- 기존 `fetchJobs()` API 호출과 `handleCreateRoadmap()` 로드맵 생성 흐름은 그대로 사용한다.
- 결과 카드는 리스트 선택에 최적화된 compact 스타일로 조정하고, 선택 상태를 시각적으로 표시한다.
- 우측 상세 패널은 선택된 공고의 회사/직무/보상/스킬/비자/마감/평가 근거를 보여준다.
- 넓은 화면에서는 결과 영역 전체를 viewport 안에 고정하고, 좌측 공고 카드 리스트와 우측 상세 오버뷰가 각각 독립적으로 스크롤되게 한다.
- `xl` 미만의 좁은 화면에서는 우측 상세 패널을 숨기고 공고 카드 리스트만 보여준다.
- 필터 바는 검색을 첫 번째 행동으로 두고, `근무 국가`, `근무 형태`, `경력 수준`, `직무 분야`, `상세 필터`를 pill 형태로 제공한다.
- 상단 검색/필터 영역은 둥근 검색창과 pill 필터 row로 구성한다.
- 검색/필터 toolbar는 별도 카드로 감싸지 않고 페이지 상단에 직접 배치해 불필요한 여백과 그림자를 줄인다.
- `/jobs` 화면은 `h-screen` 기반 앱 레이아웃으로 고정해 전체 페이지 스크롤을 막고, 공고 리스트와 상세 오버뷰만 각각 스크롤되게 한다.
- 상세 패널의 하단 CTA 영역은 고정해 `원문 보기`와 `로드맵 생성` 버튼이 항상 보이게 한다.
- 상단 toolbar는 CareerLens의 `brand/night/panel` 색감과 한국어 라벨을 사용하고, 외부 서비스와 유사한 `Save Search`, `조건 공유` 액션은 제거한다.
- `/jobs` 화면만 최대 폭을 넓혀 큰 화면의 좌우 여백을 줄이고, 다른 페이지의 공통 `.lens-container`는 변경하지 않는다.

## 최종 UI 상태

- 전체 페이지는 `h-screen`과 `overflow-hidden`으로 고정된다.
- 상단에는 얇은 타이틀 영역과 검색/필터 toolbar가 위치한다.
- 넓은 화면에서는 좌측 리스트와 우측 상세 패널이 동시에 보인다.
- 좌측 리스트는 공고 카드 영역만 내부 스크롤된다.
- 우측 상세 패널은 본문만 내부 스크롤되고, 하단 `원문 보기`와 `로드맵 생성` CTA는 항상 보인다.
- 좁은 화면에서는 우측 상세 패널을 숨기고 공고 카드 리스트를 우선한다.
- 상단 toolbar 문구는 `회사, 직무, 스킬 검색`, `근무 국가`, `근무 형태`, `경력 수준`, `직무 분야`, `상세 필터`, `필터 초기화`로 정리했다.

## 검증 결과

- `npm run typecheck`로 타입 오류를 확인한다. 결과: 통과.
- 로컬 Next.js dev server에서 `/jobs` 화면을 열어 렌더링을 확인했다.
- 백엔드가 실행되지 않은 환경에서는 실제 공고 데이터 대신 `Failed to fetch` 상태가 표시될 수 있다.
