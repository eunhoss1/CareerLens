# 커리어 플래너 상세 UI 개선 요약

## 작업 대상

```text
frontend/app/planner/[roadmapId]/page.tsx
```

커리어 플래너 목록 화면, 추천 진단 화면, 백엔드 AI 생성 로직은 변경하지 않았다.

## 개선 방향

커리어 플래너 상세 화면에서 사용자가 처음 진입했을 때 다음 내용을 빠르게 파악할 수 있도록 정리했다.

- 현재 완료율
- 남은 과제 수
- 다음 추천 과제
- 주차별 과제 진행률
- 각 과제의 상태, 예상 시간, 난이도
- 산출물과 검증 기준

## 주요 변경 내용

### 1. 핵심 진행 정보 추가

상단에 오늘의 진행 상태를 보여주는 요약 영역을 추가했다.

```text
완료율
남은 과제
다음 추천 과제
```

이를 통해 사용자가 상세 화면에 들어오자마자 다음에 처리할 과제를 확인할 수 있게 했다.

### 2. 로드맵 요약 위치 유지

로드맵 요약 패널은 기존처럼 왼쪽에 유지했다.

```text
왼쪽: 로드맵 요약
오른쪽: 주차별 과제
```

전체 화면 구조는 크게 바꾸지 않고, 기존 사용 흐름을 유지했다.

### 3. 과제 카드 가독성 개선

과제 카드의 텍스트 배치를 정리했다.

- 상단에는 상태, 카테고리, 예상 시간, 난이도 배지 표시
- 본문에는 과제 제목과 설명 표시
- 오른쪽 보조 영역에는 산출물과 검증 기준 표시

문서처럼 줄글을 읽는 느낌을 줄이고, 카드 안에서 중요한 정보가 먼저 보이도록 조정했다.

### 4. 산출물/검증 기준 강조

`산출물`, `검증 기준` 라벨을 더 잘 보이도록 글자 크기, 두께, 색상을 조정했다.

과제를 수행한 뒤 어떤 결과물을 만들어야 하고, 무엇으로 완료 여부를 판단할지 더 빠르게 확인할 수 있다.

### 5. Radius 적용

헤더 영역은 제외하고 상세 본문 카드에만 radius를 적용했다.

- 진행 요약 카드
- 로드맵 요약 패널
- 주차별 과제 카드
- 개별 과제 카드

공용 UI 컴포넌트는 수정하지 않고, 상세 페이지 내부 className만 조정했다.

### 6. Hover 범위 정리

마우스를 올렸을 때 강조되는 영역은 주차별 과제 카드로 제한했다.

```text
주차 카드: hover 테두리/그림자 적용
개별 과제 카드: hover 효과 없음
```

개별 과제 카드까지 hover를 적용하면 화면이 과하게 반응하는 느낌이 있어 제거했다.

### 7. 완료 주차 체크 표시

주차별 과제 카드 왼쪽의 타임라인 마커를 상태에 따라 다르게 표시했다.

```text
완료 전: 빈 사각형 마커
완료 후: 초록색 체크 마커
```

해당 주차의 모든 과제가 완료되면 초록색 체크가 보여서, 긴 타임라인에서도 완료된 구간을 빠르게 구분할 수 있다.

## 충돌 영향

변경 범위는 아래 파일 하나로 제한했다.

```text
frontend/app/planner/[roadmapId]/page.tsx
```

다음 파일들은 변경하지 않았다.

```text
frontend/app/planner/page.tsx
frontend/lib/planner.ts
frontend/lib/applications.ts
backend/src/main/java/com/careerlens/backend/service/PlannerService.java
backend/src/main/java/com/careerlens/backend/service/PlannerTaskDraftService.java
```

따라서 dev 브랜치에서 같은 상세 페이지를 수정하지 않았다면 머지 충돌 가능성은 낮다.

## 참고

AI 플래너 생성 방식과 관련된 백엔드 로직은 이번 UI 개선에서 변경하지 않았다.

`규칙 기반` 또는 `AI 보조` 표시는 백엔드에서 저장된 `generation_mode` 값을 그대로 보여준다.
